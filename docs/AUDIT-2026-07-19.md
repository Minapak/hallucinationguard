# Audit & hardening pass — 2026-07-19

Full-repository audit of HallucinationGuard, followed by fixes for every confirmed
finding. Method: six independent audit dimensions (scorer correctness, multilingual
behavior, security/supply-chain, test coverage, docs-vs-behavior consistency,
fixture data quality), every raw finding **adversarially re-verified by reproduction**
before being accepted; 27 findings confirmed and fixed, 10 refuted (recorded below —
what was *not* broken matters as much as what was). Every design decision cites a
primary source. All quantitative claims below were reproduced by running the code in
this repository.

---

## 1. Scorer (`reference/score_v0.py`)

### 1.1 CJK tokenization was vacuous (confirmed, high)
Whitespace splitting collapses Japanese/Chinese text (*scriptio continua* — written
without word delimiters [1]) to 1–13 pseudo-tokens per KB entry vs 29–145 for other
locales. Every ja/zh "match" was a math fragment (`=`, `+`, `|β|²`); the braket
KB entry tokenized to exactly 1 token, flipping HG006 to a false MISS in both locales.
SacreBLEU documents the same collapse: Japanese scored 2.1 BLEU with the default
whitespace tokenizer vs 14.5 with morphological segmentation [2].

**Fix — Tokenizer V0.1**: runs of Han/Hiragana/Katakana become character bigrams;
Latin/Hangul behavior unchanged. Precedent: chrF scores on character n-grams precisely
because it "requires no tokenization" and is language-independent [3]; the WMT17
metrics task scored Chinese by separating characters [4]; SacreBLEU's `zh` tokenizer
uses the same hardcoded Unicode-range approach this fix uses [5]. Korean deliberately
keeps whitespace *eojeol* tokens: spacing exists in Korean, morphological splitting
(MeCab-ko [6]) would break the zero-dependency contract, and the resulting ko misses
are documented V0 naivety. Trade-off stated: honest ja/zh agreement dropped from a
vacuous 19/20 to a meaningful 18/20.

### 1.2 The misconception gate was silently inert in 6 of 7 locales (confirmed, critical)
`forbidden_claims` were English-only strings; token recall of English claims against
ko/ja/zh answers is ≈0, so the gate fired 6 times in en and **0 times in every other
locale** — while docs claimed unconditional gating. Fix: schema v3.1 localizes every
claim to all 7 locales (empirically tuned so each locale's patterns catch the recorded
answers that assert them); the scorer selects the active locale's claims; a legacy
list is accepted but documented as en-only. Post-fix the blocked set is identical
across all 7 locales: {HG002, HG023, HG031, HG039, HG051, HG073, HG115} — pinned in CI.

### 1.3 The gate was display-only (confirmed, high)
`score_fixture` printed BLOCKED but never demoted the tier, contradicting the schema's
"fails regardless of overlap". Fix: a flagged answer is never `aiValidatedAgainstKB`
(demoted to `aiWithLowConfidence`); for no-KB topics the tier remains `aiNoKBMatch`
with the block reported alongside (there is no KB answer to serve) — the schema now
states exactly this.

### 1.4 The gate was direction-blind (confirmed, high — and undocumented)
A correct answer *denying* a misconception ("Entanglement **never** transmits
information faster than light") was flagged, while HG002's recorded answer *asserting*
the same misconception in paraphrase escaped. Direction-blindness of surface metrics
is well documented [7][8][9]. Fixes, all zero-dependency:
- **NegEx-style negation cues** per locale suppress the flag when the decisive
  sentence denies the claim (Chapman et al. 2001 [10]; scope-and-cue design also in
  ConText [11]). NegEx's own numbers — sensitivity 77.8%, PPV 84.5% — are quoted in
  the docs so nobody mistakes the heuristic for a solution.
- **Pseudo-negation handling**: negatively phrased claims ("**No** further
  cryptanalysis is needed…") skip the negation guard — echoing a claim's negation is
  assertion, the pseudo-negation problem NegEx documents [10].
- **Mention-vs-assertion framing cues**: HG045's answer *warns against* "the common
  overstatement that HHL is a drop-in replacement" and was being blocked for quoting
  it. Denial-framing cues ("overstatement", "misconception", 과장, 誇張, exageración…)
  approximate the stance-detection distinction (FEVER SUPPORTED/REFUTED [12]; FNC-1
  refuting-word features [13]). HG002's paraphrase gap was closed with an additional
  en claim variant matching the answer's actual phrasing.
- Remaining limits documented: paraphrases under the 0.60 recall threshold still
  escape; a cue anywhere in the decisive sentence suppresses even when it negates a
  different clause. The literature's stronger remedies — NLI checkers (AlignScore [14],
  MiniCheck [15]), negation-aware metrics (NegBLEURT [7]) — are the M2 upgrade path.

### 1.5 `verifiedKnowledgeBase` was unreachable (confirmed, high)
`classify()` could never return the tier the schema lists first; an item with that
ground truth scored MISS even for a verbatim KB answer. Fix: verbatim service is
detected by normalized string equality — *not* `overlap == 1.0`, because one-sided
overlap also reaches 1.0 for an answer containing every KB token plus hallucinated
extras (unit-tested).

### 1.6 Robustness (confirmed, medium/low)
- `--kb` default resolved to `./kb-examples` for bare filenames
  (`Path('f.json').parent.parent == '.'`); fixed with `resolve()` (CLI-tested).
- Malformed input crashed with raw tracebacks; now caught with one-line stderr
  diagnostics and exit 1, matching CPython's documented exit conventions [16] and
  argparse's status-2 usage errors [17].
- Empty `items` no longer ZeroDivisionErrors.
- `load_kb` falls back per-field (an empty locale `detailedExplanation` no longer
  silently drops the entry into `aiNoKBMatch`), and both fallback directions
  (answer→en, KB→en) now warn on stderr — a cross-language comparison is never silent.
- KB sides tokenizing below 5 tokens warn (meaningless denominator).
- 7-column table printed with 7 widths (dead `row[6]` hack removed); truncation
  ellipsis only when truncating; production-baseline footer only for ≥20-item runs.

## 2. Fixtures

### 2.1 Stub translations validated hallucinations (confirmed, high)
Grover's es KB text was 13 tokens; HG039's hallucinated "Grover solves NP-complete
problems" answer matched 4 stopwords {'de','grover','para','y'} → 0.308 ≥ 0.30 →
**false validation**, in exactly the locales where the en-only gate was also inert.
Fix: `grover_algorithm`, `phase_s`, `entanglement`, `ml_kem` backfilled in all six
non-EN locales (`quantum_error_correction`: fr repair + ja/zh expansion) — faithful
renderings of the verified EN text, translation-pass noted in `certaintyNote`,
`verifiedBy` untouched, native-speaker review pending (M3). ml_kem's date-qualified
2024–26 standardization sentences are preserved in every locale per the repo's
dated-sentence policy. CI now enforces script-density-aware parity floors
(es/fr/de ≥ 50%, ko ≥ 40%, ja ≥ 25%, zh-Hans ≥ 18% of en chars). Post-fix, HG039
scores 0.20 (es) / 0.16 (fr) — correctly below threshold.

### 2.2 Missing diacritics in "verified" translations (confirmed, high)
Entire fr/es blocks of `grover_algorithm`, `phase_s`, `entanglement` were written
without diacritics; `quantum_error_correction` fr used **"intrigues"** (plots/schemes)
for "intriqués" (entangled). All repaired; CI now rejects es/fr paragraphs containing
zero accented characters.

### 2.3 Fixture metadata (confirmed, medium/low)
- `expected_keywords` of HG045/HG105 could never appear in their en answers
  ('condition number' vs "well-conditioned A"; 'flux qubit' vs "flux-qubit") —
  corrected to the substring standard the other 18 items already satisfy; CI enforces it.
- Five never-consumed fields (`expected_keywords`, `ai_confidence`, `key_concepts`,
  `preskill_part`, `kb_category`) are now annotated in the schema as
  production/annotation metadata and bounds-checked in CI (1–24, 0–1, category
  consistency) so silent corruption cannot pass.
- `sample_distribution` and `categories_in_sample` are now checked against actual counts.
- Recorded `ai_explanations` are declared frozen artifacts in the schema — the audit
  explicitly did **not** "correct" recorded model answers (including their pre-KPS
  terminology), since that would falsify the benchmark.

## 3. Tests & CI

- **Zero scorer tests existed**; CI ran 7 locales and checked only exit codes, so any
  agreement regression — even to 0/20 — passed. New `tests/test_scorer.py`: unit tests
  for tokenizer/overlap/classifier/gate, CLI error paths, and a pinned per-locale
  regression matrix (agreement counts, miss sets, blocked sets for all 7 locales).
- **Actions pinned to full commit SHAs** (verified live against the GitHub API on
  2026-07-19: `actions/checkout@9c091bb…` = v7.0.0, `actions/setup-python@ece7cb0…` =
  v6.3.0). Tags are mutable; GitHub's hardening guide calls full-SHA pinning the only
  immutable reference [18], OpenSSF Scorecard requires it [19], and the March 2025
  tj-actions/changed-files compromise (CVE-2025-30066) — tags retroactively moved to a
  malicious commit, 23,000+ repos exposed — is the concrete case [20]. Trade-off
  stated honestly: SHA pins don't auto-receive backported security fixes (GitHub's
  June 2026 checkout backport shipped to floating tags first [21]) — mitigate with
  Dependabot, which updates SHA + version comment together [22].
- **Explicit least-privilege token** (`permissions: contents: read`): the audit
  *refuted* the claim that the default was write (this repo post-dates GitHub's 2023
  read-only default), so this is defense-in-depth per GitHub docs and Scorecard's
  Token-Permissions check [18][19], not a vulnerability fix.
- **Version matrix 3.9 + 3.14**: README claims "Python 3.9+" but CI tested only 3.12.
  3.9 compatibility was verified on real CPython 3.9.6. Python 3.9 is EOL as of
  2025-10-31 (PEP 596 [23]); the README now says so, and the floor moves to 3.10 at M2
  per PyPA guidance on dropping versions [24].
- Concurrency cancellation added (refuted-as-defect; pure runner hygiene per GitHub
  docs [25]).
- `.gitignore` now covers `.pytest_cache/`; `CITATION.cff` gains the optional-but-
  recommended `version` and `date-released` fields (CFF 1.2.0 schema guide [26]).

## 4. Docs honesty

- README no longer implies the public sample can reproduce the 0.9083 production
  figure: the sample scores 20/20 (en) and contains none of the 11 full-set misses;
  the "two adversarial items" are now correctly described as 2 *of* 11 misses; M1
  commits to shipping at least one failing item.
- The four-tier outcome (previously "three tiers" with an unreachable fourth) is
  documented consistently across README, schema, and code.
- Per-locale results are published as a table with the regression pins, including the
  ja/zh caveat that the pre-audit 19/20 was vacuous.
- Baseline limitations and the NLI upgrade path are stated with costs in both
  directions (MiniCheck ~GPT-4 accuracy at ~1/400 cost but adds a model dependency
  [15]; semantic entropy needs no labels but multiplies inference cost [27]).

## 5. Refuted findings (kept for the record)

Adversarial verification rejected 10 raw findings, including: Korean agglutination
and punctuation-attachment as "bugs" (both are documented V0 naivety — the repo's
stated subject of measurement); the missing `permissions:` block as a vulnerability
(GitHub API confirmed this repo's default token is already read-only);
"1,680 verified fields" as misleading (the README discloses the 280-field sample in
the same document); sample `ai_explanations` using pre-KPS terminology (recorded
answers are frozen artifacts; the KPS policy is scoped to KB entries); demo
hardcoding and `sys.path` ordering (harmless in fact — stdlib modules import before
the path insert).
